LegiApp tracks the CA State Legislature for advocacy orgs/lobbyists (a separate app from the user's other project). TypeScript monorepo via npm workspaces: `packages/{db (Drizzle+Postgres), shared (zod contract + reconcile())}`, `apps/{api (Fastify REST), web (React+Vite+Tailwind, shadcn-style), ingest (ETL worker)}`.

**PIVOT — "LegiApp for Nova Ukraine" (2026-06-18):** repurposed into a focused, publishable lobbying-research tool for **Nova Ukraine**, branded "LegiApp for Nova Ukraine" (footer credits Arseniy Shafran; a must-accept research-disclaimer modal). Key changes: (1) **Multi-session data** — `pubinfo/normalize.ts` does session-scoped `DELETE WHERE session_year` (not blanket TRUNCATE) + an `active = (session_year = max)` flag, so 2021-2022 + 2023-2024 + 2025-2026 coexist (`ingest -- pubinfo 2021|2023`); legislator/bill ids embed the session. Read paths default to the current session (legislators list `active=true`, bills `session_year=max`; `?session=all` overrides). (2) **Ukraine/Foreign-Affairs tracker** `/foreign-affairs` (`GET /api/foreign-affairs`) — region tags (`packages/shared/src/foreignAffairs.ts` `FA_REGIONS`, curation-refined stems, matched over **title+digest** only, `bill_subject` source='foreign-affairs', `ingest -- foreign-affairs`); shows bills across sessions with authors + coauthors flagged **in-office vs left**, chaptered status, region chips, CSV (Bills + **Allies**), and a **legislator alignment leaderboard** (`alignmentScore`/`alignmentLevel`: authored×3 + coauthored + passed×2 → Champion/Strong Ally/Ally/Supporter). 5 Ukraine measures span sessions (SB 1328, SJR 14, SCR 89/96, ACR 128). (3) **Access code** — single shared code gates the whole site (`apps/api/src/access.ts` HMAC cookie + `onRequest` hook; `ACCESS_CODE` env, dev default `nova-ukraine-dev`; web `lib/access.tsx`). The entire **per-user system was removed** (login/register, watchlist, alerts, saved-filters; api routes/auth|watchlist|alerts + auth.ts; ingest alerts/). (4) **Contacts** — `ingest -- contacts` pulls email+phone for the 120 current members key-free from OpenStates People `data/ca/legislature/*.yml`. Bills page defaults to **active** (data-relative last-30-days, not dead/vetoed). REMAINING: focused publish snapshot (cap to FA + ~200 active), FA-alignment badge on the legislator profile, deploy bundle (docker-compose.prod + Caddy). Tests: 45.

**Core decision:** data comes from California's official **PUBINFO bulk download** (downloads.leginfo.legislature.ca.gov — no API key, includes individual roll-call votes), NOT Open States (whose CA votes/committees are flagged experimental). PUBINFO `.dat` files are tab-delimited, optionally backtick-enclosed, with a literal `NULL` token; loaded via Postgres COPY into a `raw.*` staging schema, then normalized with set-based SQL. Schema details were read from `pubinfo_load.zip` DDL — do not invent columns.

**Status (2026-06-17):** vertical slice + Round 2 shipped on real 2025-2026 data (~120 legislators, ~5k bills, ~445k vote records). Round 2 added: (1) **full-text bill search** — bill digest+body extracted from CAML XML `.lob` files (`<caml:DigestText>`, whole-doc text) into `bill.digest`/`bill.full_text` + a weighted generated `search_tsv` GIN index; API uses `websearch_to_tsquery`+`ts_rank`+`ts_headline` (fixed "Ukraine" 1→6 hits). (2) **District map** at `/map` — Census TIGER SLDL/SLDU → GeoJSON via `mapshaper` (`npm run ingest -- districts`), Leaflet+OSM, click→both reps via `@turf/boolean-point-in-polygon`. (3) **Member gallery** + party-composition bar + best-effort Senate photos (~21/40 scraped from sd##.senate.ca.gov `*headshot*` imgs).

**Windows DB gotcha:** embedded Postgres intermittently fails to fork connections ("could not reserve shared memory region, error 487"); mitigated by retry (`packages/db/src/retry.ts` `withDbRetry`, `connectClient`) + reduced shared_buffers/no parallel workers in `scripts/db.mjs`. NOT reaping — the server stays up.

**Round 3 Phase C (calendar & deadlines) shipped 2026-06-17** on a new desktop machine: new `apps/ingest/src/calendar/` adapter (`ingest -- calendar`, also wired into `runAll`) fetches the official CA **Senate deadlines ICS** (`senate.ca.gov/.../2026-senate-legislative-deadlines-calendar.ics`) via a dependency-free RFC-5545 parser (`ics.ts` `parseIcs`/`categorizeEvent`), plus curated **CA SoS** 2026 election dates (`data.ts`); atomic delete-then-insert keyed on a new `calendar_event.external_id` (+ `detail`/`source_url`, migration 0004). API `GET /api/calendar` (type/deadline/upcoming/from-to filters) + dashboard `upcomingDeadlines`; web `/calendar` page + nav + dashboard panel. All dates trace to official sources (never-invent rule). A 13-finding adversarial review was run and all but one nit fixed.

**Snapshot-import gotcha (fixed):** `npm run db:import` did `TRUNCATE RESTART IDENTITY` + `COPY` of explicit ids but never re-synced serial sequences, so the *next* `DEFAULT` insert into any serial-PK table (e.g. `ingest_run`) collided on the pkey. `apps/ingest/src/snapshot/import.ts` now runs `setval(pg_get_serial_sequence(...))` per table after COPY.

**Round 3 Phases B + D shipped 2026-06-17 (key-free):** B — committee rosters from the OpenStates **People** GitHub repo (`data/ca/committees/*.yml`, no API key; `ingest -- committees` → 1185 memberships/119 chairs) + keyword issue-area `bill_subject` tagging (`ingest -- subjects` → ~19k tags/21 subjects); the Bills subject filter + committee rosters now populate. Cross-source **reconciliation** still needs an OpenStates key (adapter `openstates/run.ts` no-ops without one). D — email+password auth (`bcryptjs` + JWT in an httpOnly cookie, `apps/api/src/auth.ts`, `@fastify/cookie`), watchlist/saved-filters/alerts CRUD (`routes/auth|watchlist|alerts.ts`, every protected route scoped to the cookie's uid), alert trigger-checker + `nodemailer` (`apps/ingest/src/alerts/`, dev console transport unless `SMTP_URL`; `ingest -- alerts [--test]`, in scheduler); web AuthProvider/Login/Watchlist + Follow/Alerts controls. New env: `JWT_SECRET`, `WEB_PUBLIC_URL`. An 17-finding adversarial review (security-focused) was run + all fixed — incl. a **blocker** (JWT_SECRET hardcoded default → refuse to boot in prod if unset/<32 chars) and adding `@fastify/rate-limit` (10/15min on auth) + `@fastify/helmet`.
